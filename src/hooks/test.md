1. Amine : Le Cadrage & Le Besoin (Phase 1)

*Objectif :* Poser les bases solides du projet pour justifier son existence.

* *Tâche A : Le Contexte et QQOQCP*
* Rédiger l'introduction sur la santé mentale au Maroc (chiffres 48,9%, 1 psy/100k hab).
* Formaliser le QQOQCP (Quoi, Qui, Où, Quand, Comment, Combien, Pourquoi) pour clarifier le périmètre.


* *Tâche B : Les 5 Pourquoi (Analyse Causale)*
* rédiger la chaîne de causalité prouvant que la plateforme est la solution racine (Problème  Pénurie  Solution Digitale).


* *Tâche C : La Note de Cadrage Synthétique*
* Assembler le tableau final qui résume tout (Titre, MOA/MOE, Dates clés, Budget global).

2. Houcine : La Stratégie & Les Objectifs (Phase 1)

*Objectif :* Définir la direction stratégique et les cibles à atteindre.

* *Tâche A : Objectifs SMARTE*
* Définir précisément les objectifs (ex: 100k utilisateurs).
* S'assurer que chaque lettre (S.M.A.R.T.E) est justifiée, surtout le E (Éthique/Données personnelles).


* *Tâche B : Analyse SWOT*
* Lister les Forces/Faiblesses (Interne à votre équipe : ex: compétence technique vs manque d'expérience médicale).
* Lister Opportunités/Menaces (Marché : ex: Plan gouvernemental vs Tabou social).

3. Soufiane : L'Analyse Fonctionnelle (Phase 2)

*Objectif :* Traduire le besoin métier en fonctions techniques claires.

* *Tâche A : La Bête à Cornes*
* Formaliser le graphique : À qui rend-on service ? (Patients) Sur quoi agit-on ? (Santé mentale).


* *Tâche B : Le Diagramme FAST*
* Construire le tableau : Fonction de Service (ex: Orienter)  Fonction Technique (ex: Filtrer par localisation)  Solution Technique (ex: Algorithme de géolocalisation).
* *Note :* C'est crucial pour justifier les choix techniques du projet.

4. Nada : La Structure du Projet (Phase 2)

*Objectif :* Découper le "Gros" projet en petits morceaux gérables.

* *Tâche A : Le PBS (Product Breakdown Structure)*
* Lister tous les **Livrables** tangibles (L'App Mobile, Le Back-Office, L'API, La Documentation, Le Contenu).


* *Tâche B : Le WBS (Work Breakdown Structure)*
* Lister toutes les **Tâches** nécessaires pour créer ces livrables (Conception, Dév Backend, Dév Frontend, Tests).
* *Conseil :* Vérifie que chaque élément du PBS a des tâches correspondantes dans le WBS.

5. Mohammed : La Planification (Phase 2)

*Objectif :* Organiser le temps et les priorités.

* *Tâche A : Matrice d'Eisenhower*
* Classer les tâches du WBS : Qu'est-ce qui est Urgent/Important (à faire tout de suite) vs Important/Pas Urgent (à planifier).


* *Tâche B : Le Diagramme de GANTT (Macro)*
* Créer le planning sur 12 ou 13 semaines.
* Définir les phases : Cadrage (S1-3), Développement (S4-10), Recette (S11-12).
* Indiquer les jalons clés (ex: Validation Prototype).

6. Marius : Budget & Risques (Phase 2)

*Objectif :* Sécuriser le projet financièrement et techniquement.

* *Tâche A : Le Budget Prévisionnel*
* Estimer les Dépenses (Serveurs, Licences, Marketing, "Salaires" fictifs).
* Trouver les Recettes (Subventions, Partenariats, Apport personnel) pour que le total soit à l'équilibre.


* *Tâche B : Analyse des Risques (Type AMDEC)*
* Identifier 3 ou 4 gros risques (ex: Fuite de données, Panne serveur, Non-adoption).
* Calculer la criticité (Gravité x Fréquence).
* Proposer une solution d'atténuation pour chaque risque.
